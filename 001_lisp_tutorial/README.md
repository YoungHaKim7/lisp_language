# (260711) **[Lisp로 가는 길: 왜 Lisp인가](<https://news.hada.io/topic?id=31310&utm_source=discord&utm_medium=bot&utm_campaign=5116>)**
- Lisp는 괄호 문법보다 **언어를 문제에 맞게 확장**하는 사고방식이 핵심이며, REPL·패키지·심볼·조건·재시작까지 익혀야 진가가 드러남
- 코드와 데이터가 모두 리스트로 표현되는 **동형성** 덕분에, 매크로는 평가 전 코드를 데이터처럼 다뤄 새 제어 구조와 문법 추상화를 만들 수 있음
- Common Lisp에 없는 `while`도 `defmacro`로 추가할 수 있지만, 함수로 흉내 내면 인자가 즉시 평가되어 `body`가 코드가 아니라 값이 되는 차이가 생김
- Lisp 개발은 실행 중인 프로세스에 코드를 계속 평가하는 **REPL 주도 개발**에 가까…
